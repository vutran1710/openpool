```go
// cmd/action-tool/unmatch.go
// Reads issue body, decrypts to get {author_match_hash, target_match_hash},
// verifies author is in the match, deletes matches/<pair_hash>.json, commits.
```

Flow:
1. Parse issue body (openpool block type `unmatch`)
2. Decrypt with operator key → `{author_match_hash, target_match_hash}`
3. Compute `pair_hash = sha256(min(a,b) + ":" + max(a,b))[:12]`
4. Verify `matches/<pair_hash>.json` exists
5. Delete the file
6. `AddCommitPush(["matches/"], "Unmatch: <pair_hash>")`
7. Close + lock issue as "completed"

Error cases:
- Match file doesn't exist → close issue as "not_planned" (already unmatched)
- Decryption fails → reject issue
- Invalid payload → reject issue

### Task 3: Pool helper — `SubmitUnmatchIssue`

**Files:**
- Modify: `internal/github/pool.go`

- [ ] **Step 1: Add `SubmitUnmatchIssue`**

```go
// SubmitUnmatchIssue creates an issue to unmatch with another user.
func (p *Pool) SubmitUnmatchIssue(ctx context.Context, myMatchHash, targetMatchHash string, operatorPubKey ed25519.PublicKey) (int, error) {
    payload, _ := json.Marshal(map[string]string{
        "author_match_hash": myMatchHash,
        "target_match_hash": targetMatchHash,
    })
    encrypted, err := crypto.Encrypt(operatorPubKey, payload)
    if err != nil {
        return 0, fmt.Errorf("encrypting unmatch: %w", err)
    }
    encBody := base64.StdEncoding.EncodeToString(encrypted)
    return p.SubmitIssue(ctx, "Unmatch Request", "unmatch", encBody, []string{"unmatch"})
}
```
